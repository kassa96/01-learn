Who are we?

We aim to create a set of tools designed to help programming learners develop their coding skills through active practice.

Today, with the resources available online, it is possible to become a developer through a wide range of free or low-cost tutorials.
YouTube, for example, allows a community of enthusiasts to share their expertise through various content. However, many learners simply watch these videos without putting the acquired knowledge into practice. Mastering a skill primarily relies on consistent practice, in a cycle that I call "doing by watching, trying, and asking".
This concept, inspired by ancestral practices, consists of:

  • Watching (Observation): Observing how experienced individuals accomplish a task helps to understand the techniques and methods employed.
  • Trying (Practice): Putting the acquired skills into practice allows one to challenge themselves, add a personal touch, and deepen their thinking.
  • Asking (Questioning): Asking questions helps to clarify doubts, solve obstacles, and better understand processes.

Today, thanks to the Internet, acquiring knowledge is becoming increasingly accessible. We want to complement this with practice by providing a set of tools focused on hands-on experience, aimed at supporting learners in their journey.

The tools we are developing

We aim to develop interactive tools to encourage active practice. Our first tool, ExtractCode, which we have developed, allows users to capture a specific area of a YouTube video, extract the code presented, and test it directly in a code editor.
With this artificial intelligence-based application, we aim to minimize the tendency to watch tutorials without testing the code.
ExtractCode is a Minimum Viable Product (MVP) that we plan to continuously improve.
We will also develop a Chrome extension to facilitate code extraction from YouTube. We rely on feedback from our users to refine ExtractCode and explore other features that will help learners go beyond simply watching tutorials.
